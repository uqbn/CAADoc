interface CATICatalogChapter

Interface to manage catalog chapters.
Role: A chapter is an object which contains :

  • Keywords (CATICatalogKeyword )

    • A keyword is like a property (dimension, material,...). The keywords are used for describing descriptions. See CATICatalogKeyword for more details.

  • Descriptions ( CATICatalogDescription )

    • A description contains a links towards an object: a fetaure, a document, an another chapter. See CATICatalogDescription for more details.

A Chapter is created by the CATICatalogChapterFactory.CreateChapter method. There are two kinds of chapters which depends on the second parameter of this CreateChapter method:
  • Chapter which contains other sub-chapters
  • Chapter which contains only components (feature,document,..). In this case, the chapter is also named a family.

o AddFamilyDescriptionsFromDesignTable
public virtual AddFamilyDescriptionsFromDesignTable( iDT)
Adds a family of descriptions from a design table associated to a CATPart document.
Role:This methods creates a set of descriptions on the current chapter in taken account of each row of a design table. This chapter must be a family, in other words a end chapter. See the CATICatalogChapterFactory.CreateChapter method.
The design table must contains the "PartNumber" header.

This method creates automatically on the family a keyword for each header of the design table. The value of each keyword on each description is the value taken in the design table.

Parameters:
iDT
The design table which has been retrieve from the CATPart document.
