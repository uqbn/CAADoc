Launching the Setup Tool

After installation of the RADE CD on a workstation, all users are provided with a default working environment enabling them to work with the C++ Interactive Dashboard within Visual Studio or with the Java Interactive Dashboard within IRAD.

This default environment however may not be adapted to the user's individual needs, and many options are open to him. These options are conditioned by the authorizations he has on his workstation, and an administrator has more possibilities than a simple user.

In order to launch the interactive Setup Tool, simply start the program called CATVBTSetup located in the RADE installation directory. This can be done by double-clicking on the product through a navigator, or by typing the name of the program (with its full path) in a simple command prompt.

The complete appearance of the Setup Tool looks like this, but appearance may vary from one configuration to the other:

Note on the left-hand side the various target applications you wish to install the products on. Currently, Eclipse/IBM Rational applications and Visual Studio are available.

The target applications condition the list of components that appear underneath. For example, the ENOVIA Designer Studio and CAA - Web Application Composer do not appear if only Visual Studio is selected. The list of components is also determined by the number of products you have installed from the RADE CD.

Please note that the Interactive Dialog Designer application should not be installed if the corresponding licence (the IDD - CAA - Interactive Dialog Designer product) is not available, elseway the CAA workspaces will fail to open with a licensing error. This can be checked using the RADE License Manager (CATVBTLicenser) utility. In case the application has been selected by error, open Visual Studio, go to the Tools/Addin Manager ... menu, uncheck the CAA V5 - Interactive Dialog Designer Addin Addin and restart Visual Studio.

Setting your Options

The following options do not require administrative privileges, and can be modified by an individual user.

Setting Default Variables

The Setting tab page enables you to customize variables used by the various interactive tools.

There are currently three categories of variables:

  • CodeGenerator variables. These variables enable you to customize values for the Code Generation wizards.
    • CATCompanyName: The generated code has a standard header. You can modify the name of the company developing the software. Default value is Dassault Systemes, but you can insert your own company name in its place when developing your code.
  • ENOVIA Publish variables. These variables enable you to prevalue the standard values needed to publish an ENOVIA customization to the database in order to avoid the individual user to know these values.
    • CATSettingPath: The setting path of the ENOVIA server installation
    • ENOVDataBaseAliasName: ENOVIA database alias name
    • ENOVDataBaseType: the type of the database. Possible values: db2 or oracle
    • ENOVDB2_HOME: Installation path of DB2 if db2 database installed
    • ENOVDBAID: Id of the ENOVIA database administrator
    • ENOVDBAPSSD: Password of the ENOVIA database administrator (this value is not kept)
    • ENOVORACLE_HOME: Installation path of ORACLE if ORACLE database installed
    • ENOVOutputDirectory: UNIX directory where generated libraries, SQL orders and metadata files are copied to facilitate deployment.
    • ENOVSimulatePublish: Default value for publish (YES for simulation, NO for NoSimulation, ie. database modification)
    • ENOVTableSpace: Name of the ENOVIA tablespace
    • TNS_ADMIN: For Oracle only. The default value is the installation path of oracle with /network/admin suffixed.
  • WebApplicationComposer
    • JAVA_HOME: Installation path for java. For example, C:\Program Files\Java\jdk1.5.0_07

Setting the Documentation Path

The documentation of the dialog boxes of the interactive tools is supplied on the CAA CD. To be able to access this documentation by pressing F1 on one of those dialog boxes, you must enter the field: Documentation installation Path. For example, if the CAA CD was installed on: E:\Program Files\Dassault Systemes\B17, enter: E:\Program Files\Dassault Systemes\B17\CAADoc.

If the CAA CD was installed on another workstation, checking the box: Use local copy of the documentation will copy the documentation in your RADE installation, thereby bypassing the network delay to access the documentation over the network. Be aware however that the documentation is copied in the RADE installation directory.

Finally, you can change the Tool Tip that appears at initialization of the RADE Interactive tool by providing your own html file and giving its location in the field Tip of the Day configuration file.

Changing the tck Server

The Tool Access tab page allows you to modify several tool initialization options.

  • tck_init Profile Path When installing the RADE CD, a tck initialization file is provided. This file initializes the tools that let you point to various software levels (called profiles). Refer to the tck documentation for full explanations. It is possible to point to tck initialization files installed with other levels of the RADE CD, but in practice this should not be necessary as an initialization file provided on level N gives you access to profiles set up on all other levels.

Building your Software on UNIX

The Tool Access tab page also enables you to build your software on a UNIX platform.

  • Activate Remote Access This check-box determines whether the interactive tools will allow you or not to build your software on a UNIX platform.
  • Location of the RADE tools on UNIX In this field, enter the directory of the UNIX path where you have installed the RADE tools. Note that you must enter the platform in the path. For example, if you installed the tools on AIX under /usr/DassaultSystemes/RADE, enter /usr/DassaultSystemes/RADE/aix_a.
  • Unix tck profile path In this field, enter the location of the UNIX tck initialization file. In most installation case, this path should start the same as the previous one. In our example, this would be: /usr/DassaultSystemes/RADE/aix_a/code/command/tck_init.

Running the Setup Tool

Once you have set all the options you required, press the Install push button to modify the options in your directory.

In the Log tab page, notice that the various steps and warnings are listed to guide you in case a step should fail. If the user does not have administrator privileges, he is warned first, but setup will continue for those operations not needing it.

Note that in some cases, a target tab page could appear. This page allows you to copy the installation of the RADE products to another location. This option requires administrative privileges because it modifies the registry and is not supported.
