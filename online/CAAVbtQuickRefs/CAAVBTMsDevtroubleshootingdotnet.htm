Microsoft Visual Studio Troubleshooting
Technical Article

Abstract

This article describes the problems you may encounter when using RADE tools in Microsoft Visual Studio and the corresponding workarounds.

  • Icons Are not Displayed Properly in the Menus
  • A Command is Missing in the Menus
  • Projects are not up-to-date
  • Licensing Error when opening a CAA workspace

Icons Are not Displayed Properly in the Menus

  1. From the Tools menu, use alternatively the following commands:
    • Remove Mkmk Add-in commands from menus
    • Remove CodeGenerator Add-in commands from menus
    • Remove SCM commands Add-in from menus
  2. Close Microsoft Visual and re-launch it.

[Top]

A Command is Missing in the Menus

  1. From the Tools menu, select Customize...
  2. In the Customize dialog box, select the Commands tab and select the Category and the command you want to add. Select Addins->Define Prerequisite Frameworks for example.

  3. Drag and drop this command to the desired menu.

[Top]

Projects are not up-to-date

If the file tree does not match the projects content, select Project->Refresh Active Project from Filetree. If you notice discrepancies between the file tree and the whole solution, select Project->Refresh Whole Solution from Filetree.

[Top]

Licensing Error when opening a CAA workspace

The following message can be issued when opening a CAA workspace: Unable to Request License component of CAA Visual Studio Integration. You may not have access to the CAA RADE licence. This can occur when the Interactive Dialog Designer Addin whereas the corresponding licence is not available.

In case the Addin has been installed by error, open Visual Studio, go to the Tools/Addin Manager ... menu, uncheck the CAA V5 - Interactive Dialog Designer Addin Addin and restart Visual Studio.
