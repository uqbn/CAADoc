With this popup you can migrate a workspace.

 

If you have developed in a workspace on a CAA Release N level, and wish to migrate your code to a new CAA Release N+x level, you must migrate your workspace. The first thing to do is to create a new workspace, as described above. Select a new location for the new workspace, as well as the tool level corresponding to the CAA Version level you are targeting. After this is done, appears a choice of possible options. Select "Migrate a workspace" and press "Finish".

In Target Level, select the CAA release to which you want to migrate your workspace.

In Origin Workspace, enter or select the pathname of the workspace to migrate. If a UNIX connection is enabled and if the CAA release you select in Target Level is less than V5R11, you must also enter the pathname of the workspace's replica on UNIX in Origin Workspace UNIX.

The new workspace will then be populated with the code from the origin workspace. You need to set your prerequisites to point to the new CAA code and build your code.
