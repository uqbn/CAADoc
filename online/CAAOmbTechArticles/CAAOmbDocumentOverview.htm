A description of standard document types
Technical Article

Abstract

This article describes the existing standard document types provided by Dassault Systemes and explains their characteristics and usage environment.

  • The Part Document
  • The Product Document
  • The Drawing Document
  • The Material Document
  • The Catalog Document
  • The Analysis Document
  • The Process Document

The following document types provided by Dassault Systemes are all structured standard documents.

The Part Document

A Part Document has a .CATPart suffix. It is mainly used during the design process. The Part Document is always stored as an independent unit, whether on disk or in data base.

The Part Document is used as the default document type for the following workbenches:

  • Mechanical Design: Part Design, Sketcher, Sheet Metal Design, Wireframe & Surface Design, Functional Dimensioning & Tolerancing.
  • Shape: Free Style, GSM Debug, Digitized Shape Editor, Generative Shape Design.
  • Equipment & Systems: Circuit Board Design.

[Top]

The Product Document

A Product Document has a .CATProduct suffix. It is mainly used to assemble Product or Part documents in a structured, coherent system of sub-products and/or parts.

When a Product Document is saved on disk, links are automatically created to the external product and/or part documents that make it up. When it is loaded in memory, these links are resolved by loading all of the external product and/or part documents pertaining to it as well.

Using ENOVIA, a Product Document stored in data base can be exploded: each sub-product of the structure can be saved elsewhere, thus modifying the original product structure. During the explode, constraints are also stored as separate entities.

It is possible to reposition parts within the Product Document in order to assemble the same part in different locations of the system being designed.

Constraints can also be put on parts being assembled so that they become intrinsically tied to other parts.

The Product Document allows extensions to be created in order to add new behaviors to different objects.

The Product Document is used as the default document type for the following workbenches:

  • Infrastructure: Product Structure, Knowledge Advisor, Knowledge Expert, Generative Knowledge, Product Engineering Optimizer, Rendering.
  • Mechanical Design: Assembly Design, Structure Design.
  • AEC Plant: Plant Layout.
  • Digital Mockup: Navigator, Space Analysis, Kinematics, Fitting, Plant & Ship Review, Optimizer, Tolerancing Review.
  • Equipment & Systems: Systems Space Reservation, Systems Routing, Electrical Wire Routing.
  • DELMIA Infrastructure: Digital Manufacturing ReView.

The Drawing Document

A Drawing Document has a .CATDrawing suffix. It is used in the Mechanical Design working environment for Drafting, for which it is the default document type.

[Top]

The Material Document

A Material Document has a .CATMaterial suffix. It is used in the Infrastructure working environment in the Material Library workbench, for which it is the default document type.

[Top]

The Catalog Document

A Catalog Document has a .catalog suffix. It is used in the Infrastructure working environment in the Catalog Editor workbench, for which it is the default document type.

[Top]

The Analysis Document

An Analysis Document has a .CATAnalysis suffix. It is used in the Analysis & Simulation working environment in the Advanced Meshing Tools and Generative Structural Analysis workbenches, for which it is the default document type.

[Top]

The Process Document

A Process Document has a .CATProcess suffix. It is used in the Digital Process for Manufacturing working environment in the Process Verification, Assembly Process Simulation and Process Planner workbenches, for which it is the default document type.
