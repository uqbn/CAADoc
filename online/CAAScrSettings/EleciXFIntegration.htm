EleciXFIntegration Setting Repository
Quick Reference

EleciXFIntegration

Role: Interface for interaction with external data

Detailed role:It gives an interface to interact with external data i.e. iXF data repository path, working catalog path

The EleciXFIntegration repository has the following characteristics:

  • Primary Tab Page: /Equipment & Systems/Electrical Process Interfacing
  • Repository File: EleciXFIntegration
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

EleciXFIntegration setting repository attributes:

  • EnableExtAppIntegration
    Role: Enables external system interfacing
    Detailed role: Allows you to work with external systems
    • Type: integer
    • Default value: 1
    • Default Value Role: Enable external system interfacing
    • Values:
      0 Disable external system interfacing
      1 Enable external system interfacing
  • IdentifierAttribute
    Role: Sets the identifier to the 'instance name' attribute
    Detailed role: Onle set the identifier to the 'instance name' attribute, when using ENOVIA V5 VPM
    • Type: integer
    • Default value: 0
    • Values:
      0 Do not set the identifier to the 'instance name' attribute
      1 Set the identifier to the 'instance name' attribute
  • WorkingCatalog
    Role: Defines the electrical working catalog
    Detailed role: Defines the electrical working catalog that will be used for the placement of devices
    • Type: string
    • Default value:
  • iXFRepository
    Role: Defines the location of the xml files
    Detailed role: Defines the location of the xml files
    • Type: string
    • Default value:
  • iXFRepositoryinEnovia
    Role:
    • Type: integer
    • Default value: 2
    • Values:
      1 Use database
      2 Use folder option to specify the xml path
