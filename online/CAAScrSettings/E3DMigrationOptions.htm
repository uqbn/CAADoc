E3DMigrationOptions Setting Repository
Quick Reference

E3DMigrationOptions

Role: Options to set while migrating data from V4 to V5

Detailed role:Options to set while migrating data from V4 to V5

The E3DMigrationOptions repository has the following characteristics:

  • Primary Tab Page: /General/Compatibility/Electrical
  • Repository File: E3DMigrationOptions
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

E3DMigrationOptions setting repository attributes:

  • GBNOptions
    Role: Geometrical bundle migration mode
    Detailed role: Geometrical bundle can be migrated as a component OR product document
    • Type: string
    • Default value: IdProduct
    • Values:
      "IdProduct" Migrate Geometrical bundle as a CATProduct.
      "IdComponent" Migrate Geometrical bundle as a component.
  • Option
    Role: Bundle segment migration mode
    Detailed role: Bundle segment can be migrated as a bundle segment OR multi-branchable document
    • Type: string
    • Default value: IdBranchable
    • Values:
      "IdBranchable" Migrate V4 BNSs to a V5 multi-branchable document. All the V4 BNSs of a given V4 GBN will result in a multi-branchable document: a single CATPart.
      "IdBNS" Migrate V4 BNSs to V5 bundle segments. Each V5 bundle segment will have its own CATPart document.
  • ShellOptions
    Role: Shell migration mode
    • Type: string
    • Default value: IdMigration
    • Values:
      "IdMigration" Migrate the Shells With Electrical Behavior.
      "IdGeomRep" Migrate Shells Without Electrical Behavior.
