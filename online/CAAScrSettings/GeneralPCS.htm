GeneralPCS Setting Repository
Quick Reference

GeneralPCS

Role: Setting to manage performance, capacity and scalability tuning

The GeneralPCS repository has the following characteristics:

  • Primary Tab Page: /General/PCS
  • Repository File: GeneralPCS
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

GeneralPCS setting repository attributes:

  • CheckMultiEditorUndo
    Role: Check multi-editor Undo
    Detailed role: Activating this setting prevents any unpredictable behavior in the Undo/Redo process
    if a document is open in several editors for modification purpose.
    • Type: unsigned integer
    • Default value: 1
    • Min value: 0
    • Max value: 1
    • Step: 1
  • StackFullWarning
    Role: Activation flag for stack full warning
    Detailed role: Sets the behavior when the Undo/Redo stack becomes full.
    0 means that no warning is displayed, 1 displays an easy warning, 2 displays a notify box requesting a user commitment.
    • Type: unsigned integer
    • Default value: 0
    • Min value: 0
    • Max value: 2
    • Step: 1
  • UndoMax
    Role: Maximum number of undoable global transactions per document
    Detailed role: Maximum number of commands or editions in context which can be undone for each document.
    0 means that only local transactions of the current command or edition in context are undoable.
    • Type: unsigned integer
    • Default value: 10
    • Min value: 0
    • Max value: 99
    • Step: 1
