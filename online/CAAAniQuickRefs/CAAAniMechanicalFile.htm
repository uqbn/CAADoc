The physic's dictionary inside CATIA
Quick Reference

Abstract

This article describes the way all physical types are defined inside the analysis modeler. The intent of this dictionary is to define a physics independent of the solver that is used. This allows to have standard definition for data exchange and also coupling of analysis kind.

  • The "Mechanical File"
  • The declaration of new physical types : PROCESS       NEW R18

All the physical types are defined in the "Mechanical" File. This file is delivered with standard CATIA installation in XML language (extensible markup language). For general rules and recommendation about XML consideration inside CAAV5, refer to [1].

For development purpose, you can extend this file. In order to be completed, the standard dictionary is delivered inside the CAA analysis documentation under the "CAAAniTechArticles.doc\PhysicalTypes" directory, with an HTML view. Remember that usage of this physical types dictionary extension functionality is limited to development phases. The license associated to development tools is requested while loading the extended XML file.

The steps to respect to extend the physical types dictionary are the following:

  • Copy the Mechanical.xml file from the documentation to your development framework in the "CNext\reffiles\SIM" directory. You can also use the XML version that is provided in the CATIARuntime installation. This is located in
    Windows InstallRootDirectory\intel_a\reffiles\SIM\CATIALevel\
    Unix InstallRootDirectory/intel_a/reffiles/SIM/CATIALevel/

    where InstallRootDirectory is the directory where the CAA CD-ROM is installed, CATIALevel and is the version level you have installed. As example, CATIALevel is set to 51700 for V5 Release 17.

  • Add a new XML element PhysicalDescriptionPackage in the XML element PhysicalDescriptionDirectory.
  • Add your new physical types in the PhysicalDescriptionPackage you have created before.
  • Update the CATIA "Run time view", with the mkCreateRuntimeView command.
  • This new file will be loaded if  CATIA detects that you have a development environment, it means that we will control that you have a development licence installed. Two ways can be used:
    • Copy the content of the directory "ProductIC" of your RADE installation to the "ProductIC" directory of the CATIA installation without replacing existing files.
    • Or create a new workspace in which you copy the os/code/ProductIC of the CAA RADE and complete your prerequisite by using the mkGetPreq command with this new workspace.
  • Activate CDC licence in the  Tools/Option Licencing Menu.

Once your validation is done, send your new physical types to Dassault Systèmes in order to integrate them inside the standard dictionary with the next version of CATIA. For this, you must deliver the request in CAA V5 Web Forum @www.caav5.com. Remember that new physical types must be understood by other solvers.

Note that since R14 Release, default installation wizard propose you to install CAA tools and CATIA Runtime in the same directory. The drawback, if you accept this, is that you will no more be able to activate the CDC license in Tools/Option. So, prefer to install in separate directories or is your installation is already done, rename the file os\reffiles\CAARADEProducts.txt that is used to filter CAA product in Tools/Option.

Also, os is the directory corresponding to your operating system.

Process for declaration of new physical types on the coming release

If you need to declare new physical types in the official Mechanical File on the next level of release, you have to follow the following process :

  1. Fill in the following spread sheet where you will have to describe the new physical types, a scenario of use and some other information.

        The template NewPhysicalTypeRequest.xls is located in CAAAniQuickRefs.doc directory.   

  2. Send it to the hotline or to the CAA SIMULIA V5 Partners Support.
