How to install RSx
Technical Article

Abstract

This article is intended for people administrating the RSA 7.0 and RADE interactive products.

  • Before You Start
  • Launching the Configuration Tool

Before You Start

Make sure you have checked ENOVIA Designer Studio and CAA - Web Application Composer in the CAA V5 Development Environment Setup window. To find out more, see customizing the RADE Interactive Tools.

Launching the Configuration Tool

From RSA 7.0 onward, no distinction is made between the different RSx products meaning that there is only one installation process. To perform this installation, you need to create an Eclipse extension. To do so, proceed as follows:

  1. Start RSA 7.0.
  2. From the Help menu, select Software Updates->Manage Configuration. The Product Configuration window is displayed.
  3. Click Add an Extension Location.

    The Browse For Folder dialog box is displayed.

  4. Select the extension location.
  5. Click Yes when asked if you want to restart Rational Software Architect.
  6. From the Help menu, select Software Updates->Manage Configuration. The Product Configuration window is displayed. A new eclipse node has been added to the configuration.

32 bits Server support on Windows 2008 64 bits platform

It is possible to run a 32 bits ENOVIA Server on Windows 2008 64bits platform. By default, on 64 bits platforms, CAA tools (mkmk, mkGetPreq, etc...) are setup to manage 64 bits runtime and buildtime: paths are configured to reference win_b64 directories.

Therefore, if a 32 bits ENOVIA Server is installed on a 64 bits platform, it is necessary to set the _MkmkOS_BitMode variable to 32 before launching RSx and using the DMC product.

For more information, refer to dedicated article: CAA V5 64-bits Operating System Support
