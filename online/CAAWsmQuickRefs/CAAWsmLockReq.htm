adl_lock_req

Locking promotion
Quick Reference

Abstract

The adl_lock_req command locks the promotion to the current workspace.

  • Synopsis
  • Usage
  • Options
  • Examples
  • References

Synopsis

adl_lock_req  [-tree ws_tree_name...] [-h | -help | -?]
 

Usage

A promotion (see the [1] adl_promote command) takes into account all modifications since the last synchronization.

This command forbids the request for promotion for the child workspaces of the current workspace.

To unlock this workspace, run the [2] adl_unlock_req command.

[Top]

Options

adl_lock_req accepts the following options:
 

-h|-help|-? Help of the command
-tree ws_tree_name... To reduce the effect of the command to a subset of trees.

Examples

Locks all the promotion of the “MyProjectWs” workspace:

adl_ch_ws MyProjectWs
adl_lock_req
