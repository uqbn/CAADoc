adl_unlock_db

Unlocking a database
Quick Reference

Abstract

The adl_unlock_db command unlocks a SCM database. 

  • Synopsis
  • Usage
  • Options
  • Examples
  • References

Synopsis

adl_unlock_db database_name... [-no_check_connect] [-h | -help | -?]

Usage

For a given site, the site's administrator will manage the file contents servers and the central database. See the [1] adl_monitor and [2] adl_file_server  commands.

To manage the central database, the adl_unlock_db command will unlock the database if necessary..

[Top]

Options

adl_unlock_db accepts the following options:
 

-h | -help | -? Help of the command
database_name ... Name of databases to unlock
-no_check_connect The connection to the database is not checked.

[Top]

Examples

Unlock the "DatabaseName" database:

adl_unlock_db DatabaseName

 

Unlock the "DatabaseName" database, without any control:  

adl_unlock_db DatabaseName -no_check_connect
