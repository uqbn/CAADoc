Creating a tree
Quick Reference

Abstract

The adl_chk_site command checks consistency between data in the SCM databases and file server.

  • Synopsis
  • Usage
  • Options
  • Examples
  • References

Synopsis

adl_chk_site -check_cont_file [-db|-database database...] [-h | -help | -?]

Usage

After the restoration of a SCM database or file server data, you can use this command to check the consistency of the SCM data.

In this first version, this command checks  that all file contents declared in all site databases or in specified databases exist in contents servers.

[Top]

Options

adl_chk_site accepts the following options:  

-h|-help|-? Help of the command
-check_cont_file Check the contents servers
-db database... Check the specified databases

[Top]

Examples

Check all file contents declared in SCM databases exist in files servers:  

adl_chk_site -check_cont_file
Checking the file contents of the database ADL2 with the contents server carterdsy:23500
Command successful
