Synopsis

Usage : adl_gui [-chg [ object | change ]] | [-ws [ ws_name ]] [-h]
 

[Top]

Usage

This command launch the SCM federate graphic tool.

[Top]

Options

adl_ch_resp accepts the following options:

-h Help of the command
-chg Launch change history
object Path or name of an existing SCM object, or identifier prefixed by 'soid:'
change Identifier prefixed by 'chgid:'
-ws Launch workspace history
ws_name Name of the workspace

[Top]

Appearance

By default, you are positioned in the workspace from which you launch this tool.

But, you can change it with the menu "Open workspace".

[Top]

Open a workspace

Actually, be positioned in a workspace means only that you could use "Import" button to import changes to this workspace.

This is also the workspace name entered by default in workspace history tool.

 

You can enter a new workspace name, click "OK" and choose an image path. Only images available on your platform (UNIX or WINDOWS) are proposed. 

 

Otherwise, you can also click on the "Browse" button.

 

You can open a workspace in two different ways : 

  • enter a workspace name and select its tree
  • enter a workspace tree name and select a workspace belonging to this tree

Use tabs to choose your preferred search.

Note 1 : Reset button erase trees and allow to enter a new workspace name.

Note 2 : When the check box named "Automatic expand" is selected, all the workspaces to leaf nodes are displayed in a double-click on workspace tree name.

 

[Top]

Enter a workspace name

In this tab, you have just to enter a workspace name.

Maybe, a workspace name is already entered. This name is the workspace name from which you launched the federate tool by the command adl_gui (by default, or else this one entered in the federate tool).

If not, you have first to enter a workspace name and click on the "OK" button and choose an image path.

 

[Top]

Enter a workspace tree name

In this tab, you have to enter a workspace tree name and then you have to enter a workspace name.

First, enter a workspace tree name and click "OK". A tree appear with a workspace tree (the one you entered) and its workspaces.

Then, double-click on a workspace name.

Finally, validate by clicking on the "OK" button and choose an image path.

Note 1 : Reset button erase trees and allow to enter a new workspace name.

Note 2 : When the check box named "Automatic expand" is selected, all the workspaces to leaf nodes are displayed in a double-click on workspace tree name.

Others buttons

Others buttons correspond to SCM tools.

  • "Import" application imports changes from a workspace to another without use the standard flow (promote, collect, synchronize). It's a transverse flow. For more informations, click here.
  • "Changes history" application shows who, when and what changes and merges are made on a software object. For more informations, click here.
  • "Workspaces history" application shows how live workspaces and flows between them. For more informations, click here.
