adl_ls_contents_server

Listing contents server
Quick Reference

Abstract

The adl_ls_contents_server command lists all the contents server for a given site. 

  • Synopsis
  • Usage
  • Options
  • Examples
  • References

Synopsis

adl_ls_contents_server [-program [-sep | -separator char]] [-out file_path]  [-h | -help | -?]

Usage

For a given site, the site's administrator will manage the file contents servers.

This command displays the names of the existing contents server registered in the central database.

To register a new contents server, run the [1] adl_mk_contents_server command.

For more information on the File Contents Server, see the [2] File Server documentation.

The "-program" option displays information in a specific output mode. For more details, see the article explaining this option.

 

[Top]

Options

adl_ls_contents_server accepts the following options:
 

-h|-help|-? Help of the command
-program Display for programs: identifier, case name.
-sep char Separator between each field for programs (space " " by default)
-separator char Separator between each field for programs (space " " by default)
-out file_path Path of the file where the results are stored.

Examples

Display all the file contents server registered in the database:

adl_ls_contents_server Identifier Host name Port number Description ---------------------------------------------------------------------------------------------------------- CKEz2R5cn9+3+++-UE8n algue 23451 CKbJvM0sjRY0+++sTE+1 ange 23500 FS12 CKq4bVDfLsg3+++-GU6- carter 5 CBSTrKY9BkM1+++eJ++1 cologne 23500 FS11

 

Display all the file contents server registered in the database: in a program mode, with the character "|" used as separator:  

adl_ls_contents_server -program -sep "|" CKEz2R5cn9+3+++-UE8n|algue|23451|\0 CKbJvM0sjRY0+++sTE+1|ange| 23500|FS12 CKq4bVDfLsg3+++-GU6-|carter|5|\0 CBSTrKY9BkM1+++eJ++1|cologne|23500|FS11
