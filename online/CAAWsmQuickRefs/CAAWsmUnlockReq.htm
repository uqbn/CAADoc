adl_unlock_req

Unlocking promotions
Quick Reference

Abstract

The adl_unlock_req command unlocks the promotions to the current workspace.

  • Synopsis
  • Usage
  • Options
  • Examples
  • References

Synopsis

adl_unlock_req  [-tree ws_tree_name...] [-h | -help | -?]
 

Usage

A promotion (see the [1] adl_promote command) takes into account all modifications since the last synchronization.

This command allows the forbidden requests (locked with the [2] adl_lock_req command)  for promotion of the child workspaces of the current workspace.

[Top]

Options

adl_unlock_req accepts the following options:

-h|-help|-? Help of the command
-tree ws_tree_name... To reduce the effect of the command to a subset of trees.

Examples

Unlocks all the promotion of the “MyProjectWs” workspace:

adl_ch_ws MyProjectWs
adl_unlock_req
