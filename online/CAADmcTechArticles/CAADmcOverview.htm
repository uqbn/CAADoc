Overview

DMC is a tool that enables architects and database administrators to customize PLM data models modifying their . metadata files without coding. The customization process is performed using Rational Software Architect and is broken down into 3 main steps.

  • Defining the Environment
  • Defining the UML Data Model
  • Publishing the Data Model

[Top]

Defining the Environment

The environment definition consists in:

  • Create a new UML project
  • Create a new workspace defining its location and name
  • Specify the tool level
  • Define the server hostname, login name and workspace location (For server installed on UNIX)
  • Define CAA V5 prerequisites

[Top]

Defining the UML Data Model

The definition of the UML data model consists in:

  • Defining the PLM modeler to be customized (Product Structure Configured, …)
  • Customizing existing objects creating new user defined attributes and indexes. Once created, the new objects are automatically represented in the UML standard using class diagrams.

Publishing the Data Model

The publishing process consists in:

  • Publishing the customization
  • Updating the PLM domain definition file with the new customization
  • Generating and importing updated security rules, taking the extended object into account
  • Generating SQL orders to update the database
  • Executing SQL orders

The publish tool generates:

  •  *.param and SQL order to update the database
  •  *.custo file to describe DefaultMask
  •  SearchEntitiesDefinition.xml to customize the Search CATLet
  •  ActionRules.xml to describe Rules in ChangeManagement
  •  VPMLightXNode.xml (only for Product Structure customization) for ENOVIA Runtime performances.
  •  VPMAdmin.adm with the new customized modeler.
  • *.CustomEnv.sh (only use for RADE tools in global process of customization)
